Initialization Process

After the loading of OpenUI5 runtime is finished, the initialization process prepares the runtime for configuration and use of OpenUI5.

During the initialization of OpenUI5 runtime, the following steps are executed:

  1. The jQuery plugins, which are mainly located in the jQuery.sap namespace, provide fundamental functionality of OpenUI5, such as the modularization concept, a small logging framework, performance measurement, and so on.

  2. If not already available, the global object sap is created.

  3. The sap.ui.core.Core class is executed with all its dependencies.

  4. The runtime configuration is determined from different sources.

  5. All libraries and modules declared in the configuration as well as their dependencies are loaded.

  6. For each loaded library, the style sheet of the configured theme is added to the page.

  7. When all libraries are loaded and the document is ready, the initEvent of the core is fired and all registered handlers are executed.


In this section:
    Loading of Additional Resources During Bootstrap
    During bootstrap, the OpenUI5 runtime loads and interprets additional resources for the control libraries that are loaded.
    Dynamic Loading of Libraries
    OpenUI5 provides the sap.ui.getCore().loadLibary() method to load libraries at runtime in addition to the libraries declared in the runtime configuration.
