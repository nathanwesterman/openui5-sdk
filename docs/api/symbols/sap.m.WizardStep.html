Class sap.m.WizardStep

extends Control

The WizardStep control is an aggregation of the Wizard control. This Control is supposed to be used only inside the Wizard, and should not be used as standalone. It aggregates the contents, and gives the developer the ability to validate, or invalidate the step.


Since: 1.30.

new sap.m.WizardStep(sId?, mSettings?)
Constructor for a new WizardStep.

Accepts an object literal mSettings that defines initial property values, aggregated and associated objects as well as event handlers. See sap.ui.base.ManagedObject for a general description of the syntax of the settings object.

The supported settings are:

  • Properties
    • title : string (default: )
    • validated : boolean (default: true)
  • Aggregations
    • content : sap.ui.core.Control[]
  • Events
    • complete : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]
    • activate : fnListenerFunction or [fnListenerFunction, oListenerObject] or [oData, fnListenerFunction, oListenerObject]

In addition, all settings applicable to the base type sap.ui.core.Control can be used as well.

Parameters:
{string}sId? id for the new control, generated automatically if no id is given
{object}mSettings? initial settings for the new control
